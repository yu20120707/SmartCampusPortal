# Epic

Stand up the V1 delivery baseline for SmartCampusPortal so the next implementation task can build a real campus-facing product slice on top of the current RuoYi scaffold.

The V1 epic is intentionally narrow:

- portal aggregator for multi-role entry
- academic data experience for students
- teaching visibility for teachers
- dashboard visibility for leaders

The epic explicitly defers card, payment, OA, and broader student affairs workflows to later phases unless a dependency is discovered that blocks the portal launch.
