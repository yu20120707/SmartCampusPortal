# 性能优化

## 目的

本项目性能优化的方向和常用手段。

## 数据库优化

- 合理使用 MyBatis 分页（`PageHelper`，RuoYi 已集成）
- 为常用查询字段建立索引（如 `student_no`, `user_id`, `course_id`）
- 避免 N+1 查询，使用 MyBatis 关联查询或 `@One`/`@Many` 懒加载
- 统计分析类查询（领导大屏）使用单独的统计 SQL，避免影响业务表

## 缓存策略

RuoYi 已集成 Redis，可以用于：

| 缓存内容 | 策略 |
|---------|------|
| 系统公告 | Redis 缓存，公告变更时清除 |
| 用户权限/菜单 | RuoYi 已默认缓存 |
| 门户聚合数据 | 按角色缓存，设置过期时间（如 5 分钟） |
| 一卡通/缴费 | 实时查询数据库，不使用缓存（数据更新频繁） |

## 前端性能

- 路由懒加载（RuoYi 已配置）
- ECharts 图表按需加载
- 列表页延迟加载（分页）

## 注意事项

- 不要过早优化，先保证功能正确
- 性能问题通过实际测量（JMeter / 浏览器 Performance Tab）确认后再优化
- Redis 缓存不要缓存大量用户个性化数据
